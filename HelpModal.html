Arena Data Center Manager - Help Guide

Your complete guide to managing datacenter layouts with Arena PLM

Welcome! Let's Get You Started

Arena Data Center Manager turns a Google Sheet into a full datacenter layout design tool connected to Arena PLM. The spreadsheet is the layout: each rack gets its own sheet tab, an overview sheet is the floor-plan grid, and everything syncs bidirectionally with Arena PLM records.

The Three-Level Hierarchy

POD (Level 0) — The datacenter hall or zone — top-level Arena item
  ├— Row (Level 1) — A physical row of racks — child of POD in Arena BOM
     ├— Rack (Level 2) — An individual rack config sheet — child of Row in Arena BOM
Terminology is configurable! "POD" could be "Hall", "Row" could be "Aisle", "Rack" could be "Cabinet" — whatever fits your organization. Configure this via Setup → Configuration → Configure Type System.

Step 1: Connect to Arena

1 Open the connection dialog
Arena POD → Setup → Configuration → Configure Arena Connection

Enter your Arena email, password, and workspace ID. The dialog auto-tests your credentials on save and shows success or error.

2 Verify the connection
Arena POD → Setup → Configuration → Test Connection

A successful test shows your workspace info, item count, and category count along with a welcome message.

Tip: Your credentials are stored securely in Google's UserProperties service and are only accessible by this spreadsheet. They are never shared.

Step 2: Run Quick Setup or Customize

3 Apply defaults or customize
Arena POD → Setup → Configuration → Quick Setup (Default Config)

Quick Setup applies neutral defaults and offers to open the full configuration wizard if you want to customize terminology, colors, columns, and BOM levels.

Step 3: Create Your First Overview Layout

4 Create an overview grid
Arena POD → Create Layout → New Overview Layout

You'll be asked for a name, the number of rows (1-50), and positions per row (1-50). This creates a grid sheet representing your datacenter floor plan.

Step 4: Create Rack Configurations

5 Create rack config sheets
Arena POD → Create Layout → New Rack Configuration

Give each rack a name, then choose to link it to an existing Arena item (fetching its BOM) or start with a placeholder. Use the Item Picker sidebar to add components to its BOM.

Step 5: Place Racks in the Overview

6 Link racks to the overview grid
Arena POD → Create Layout → Autolink Racks to Overview

Automatically populates your overview grid with clickable hyperlinks to all rack config sheets. Or use the Rack Picker sidebar to place racks one at a time.

Step 6: Push to Arena

7 Publish the full POD structure
Arena POD → BOM Operations → Push POD Structure to Arena

The 3-step wizard creates the complete POD → Row → Rack hierarchy in Arena PLM. See the Publishing tab for full details.

Create Layout

New Rack Configuration
Arena POD → Create Layout → New Rack Configuration

Creates a brand-new rack configuration sheet. Prompts for a name, then asks whether to link to an existing Arena item or use a placeholder. If linking to Arena, you enter an item number which is validated and fetched via the API, optionally pulling its BOM. The new sheet includes a metadata row (PARENT_ITEM), header row, and BOM data. It is registered in the History tab and given a blue cascading tab color.

New Overview Layout
Arena POD → Create Layout → New Overview Layout

Creates an overview grid sheet representing a datacenter floor plan. Prompts for a name, number of rows (1-50), and positions per row (1-50). The sheet is structured with a blue title bar, "Row Item" column, and position headers (Pos 1, Pos 2, ...). The tab gets a green color. This is the bird's-eye view of your layout.

Clone Existing Rack
Arena POD → Create Layout → Clone Existing Rack

Opens the Rack Picker sidebar in "clone" mode. Browse or search existing rack config sheets, select one to clone, and provide a new item number and name. The cloned sheet is a full copy including all BOM rows, created as a PLACEHOLDER ready for customization.

Load Arena Item as Template
Arena POD → Create Layout → Load Arena Item as Template

Opens the Rack Picker sidebar in "template" mode for the "150% BOM" workflow. Fetch an Arena item's full BOM, preview all components, and select which ones to include. Creates a new PLACEHOLDER rack with a yellow reminder row prompting you to trim down before pushing.

Clone This Rack
Arena POD → Create Layout → Clone This Rack

Context-sensitive shortcut that only appears when you are viewing a rack config sheet. Instantly clones the current rack after prompting for a new item number and name. Fastest way to duplicate the rack you're already looking at.

Autolink Racks to Overview
Arena POD → Create Layout → Autolink Racks to Overview

Populates the overview grid with clickable hyperlinks to all rack config sheets in the spreadsheet. Each cell displays the rack's Arena part number (e.g., "100-0042") as the link text so the POD push wizard can match racks by number. Run this after creating your rack sheets to fill in the overview grid automatically.

Item & Rack Pickers

Show Item Picker
Arena POD → Show Item Picker

Opens a sidebar with 3 tabs: Items (browse/search all Arena items with infinite scroll), Filters (filter by category, lifecycle phase, favorite categories), and Quantities (view item usage counts across sheets). On a rack config sheet, inserting an item writes a full BOM row (qty, number, name, category, notes). On other sheets it inserts the item number as a hyperlink. Items with pending change orders show a red ECO badge.

Show Rack Picker
Arena POD → Show Rack Picker

Opens a sidebar with 4 tabs: Racks (browse all rack config sheets, place as hyperlinks in overview), From Arena (browse Arena items and auto-create rack sheets with BOM), Templates (clone existing racks or load Arena item BOMs as templates), and Quantities (view rack placement counts by scope). Use this to populate your overview grid or create new racks from Arena data.

Arena POD → BOM Operations → Create Consolidated BOM

Scans the overview sheet for all rack placements, gathers each rack's BOM, and generates a "Consolidated BOM" sheet with all items aggregated by total quantity. Stage 1 handles racks, Stage 2 handles rows, Stage 3 is POD creation and final review. Includes animated progress, live step logging, and a completion screen with an "Open POD in Arena" button. See the Publishing tab for full details.

Arena POD → BOM Operations → Check All Rack Status (vs. PDM)

Compares every local rack BOM against Arena's current BOM data. Updates each rack's status: Synced Arena Modified Locally Modified Placeholder Error. Shows a summary dialog with counts by status.

View Rack History
Arena POD → BOM Operations → View Rack History

Navigates to (or creates) the Rack History tab and opens the History Filter sidebar. Shows all historical sync events with filtering by rack, status, and date range. The History tab is a protected audit trail of every rack creation, status change, sync, and push event.

Creating Racks and Layouts

This section covers the complete workflow for building a datacenter layout: creating overview grids, building rack configurations, populating BOMs, and linking everything together.

Creating an Overview Layout

The overview is a grid sheet representing your datacenter floor plan. Each cell can hold a rack placement.

1 Create the overview
Arena POD → Create Layout → New Overview Layout

Enter a name (e.g., "DC-East Floor 2"), number of rows (1-50), and positions per row (1-50).

2 Understand the grid structure
  • Row 1: Blue title bar with the overview name (merged across all columns)
  • Row 2: Blank spacer
  • Row 3: Headers — Column A is blank, Column B is "Row Item", Columns C+ are "Pos 1", "Pos 2", etc.
  • Data rows: Row numbers in Column A, Row Item links in Column B (filled after POD push), rack placements in Columns C+
The overview tab gets a green color to distinguish it from rack config tabs (blue) in the sheet tab bar.

Creating Rack Configurations

There are four ways to create a rack config sheet. Choose the one that fits your workflow:

Option A: New Rack from Scratch

Arena POD → Create Layout → New Rack Configuration
  1. Enter a rack name
  2. Choose to link to an existing Arena item number or start with a placeholder
  3. If linking to Arena: enter the item number, the tool validates it via API and optionally fetches the item's BOM
  4. A new sheet tab is created with metadata row, headers, and any imported BOM data
  5. Use the Item Picker to add more components to the BOM

Option B: Clone an Existing Rack

Arena POD → Create Layout → Clone Existing Rack
  1. The Rack Picker sidebar opens in clone mode
  2. Browse or search your existing rack config sheets
  3. Select the source rack, enter a new item number and name
  4. A full copy is created including all BOM rows, set as PLACEHOLDER status
  5. Modify the clone as needed for your variant
Quick Clone: When you're already viewing a rack, use Arena POD → Create Layout → Clone This Rack for instant duplication without opening the Rack Picker.

Option C: Load Arena Item as Template (150% BOM)

Arena POD → Create Layout → Load Arena Item as Template
  1. The Rack Picker sidebar opens in template mode
  2. Browse Arena items, find one with a comprehensive BOM (all possible options)
  3. Preview the full component list and select which ones to include
  4. A PLACEHOLDER rack is created with selected components and a yellow reminder row
  5. Delete unwanted rows to trim down to your actual configuration
What is the "150% BOM" workflow? Load an Arena item containing ALL possible options (e.g., 50 component variants), then trim it down to your actual needs (e.g., 15 components). Much faster than adding items one by one.

Option D: Create from Arena via Rack Picker

Arena POD → Show Rack Picker → From Arena tab
  1. Browse Arena items in the "From Arena" tab
  2. Click "Place Item (Auto-Create Rack)" to create a new rack config sheet from an Arena item
  3. The tool creates the sheet and automatically pulls the item's BOM

Status Tracking & History

Every rack configuration has a sync status showing whether it matches Arena PLM. The tool provides comprehensive status checking and a full audit history of all events.

Understanding Rack Status

Each rack tab displays a color-coded status indicator:

SYNCED (Green) Local BOM matches Arena exactly. Everything is in sync.
ARENA MODIFIED (Orange) Someone changed the item in Arena since you last synced. Arena has newer data.
LOCALLY MODIFIED (Blue) You've made local edits to this rack's BOM that haven't been pushed to Arena yet.
PLACEHOLDER (Grey) This rack was created locally and has never been synced with Arena. It will be created in Arena during the next POD push.
ERROR (Red) The last sync operation for this rack failed. Check the History tab for error details.

Checking Rack Status

Arena POD → BOM Operations → Check All Rack Status (vs. PDM)

This operation compares every local rack BOM against Arena's current data:

  1. Each rack's BOM is fetched from Arena and compared with the local sheet data
  2. Status indicators are updated on every rack tab
  3. A summary dialog shows counts by status (e.g., "5 Synced, 2 Arena Modified, 1 Placeholder")
Smart Detection: The system distinguishes between local edits and Arena changes by comparing BOM checksums, so you always know which direction the change came from.

Viewing Rack History

Arena POD → BOM Operations → View Rack History

Navigates to (or creates) the Rack History tab and opens the History Filter sidebar. The History tab provides:

  • Summary section: Current status of all racks at a glance with quick stats
  • Detail section: Chronological log of all events across all racks
  • Filter sidebar: Automatically opens to help you navigate by rack, status, or date

Using the History Filter

  1. Select a specific rack from the dropdown to show only that rack's events
  2. Click "Apply Filter" to filter the view
  3. Click "Clear Filter" to show all events again
  4. View quick stats showing rack counts by status at any time

Event Types Logged

The History tab records these events automatically:

  • Rack Created: New rack configuration was created
  • Status Change: Rack sync status was updated
  • Local Edit: User edited a rack BOM in the spreadsheet
  • Refresh Accepted / Declined: User accepted or declined an Arena BOM refresh
  • POD Push: Rack was included in a Push POD Structure operation
  • BOM Pull: BOM data was pulled from Arena into the rack sheet
  • Manual Sync: User manually marked rack as synced
  • Batch Check: Status check operation was run
  • Error: An operation failed
Protected Data: The History tab is protected from manual editing to maintain data integrity. All updates are made automatically by the system.

Advanced Status Tools

Arena POD → Setup → Advanced
Mark Current Rack as Synced

Manually sets the active rack's status to SYNCED and logs a MANUAL_SYNC event. Use when you've manually verified the rack matches Arena but the automatic check can't confirm it.

Validate History Integrity

Checks the History tab for structural issues: all racks have summary rows, no orphaned rows exist, all status values are valid. Shows a results dialog listing any problems found. Run this if history data seems corrupted.

Repair History Issues

Runs validation, shows discovered issues, asks for confirmation, then auto-fixes problems by creating missing summary rows and cleaning up orphaned data. Always run Validate first to see what will be changed.

Tip: Run Validate History Integrity periodically, especially after importing racks or if the history tab seems incomplete.

Publishing to Arena

Publishing creates the complete POD → Row → Rack hierarchy in Arena PLM from your spreadsheet layout. This is the primary way to push your datacenter design into Arena.

Understanding the POD Structure

The push wizard creates a three-level hierarchy in Arena:

POD (Level 0) — Top-level item for the entire datacenter zone
  ├— Row 1 (Level 1) — Contains racks in that row as BOM children
  |   ├— Rack A (Level 2) — Individual rack with component BOM
  |   └— Rack B (Level 2) — Individual rack with component BOM
  └— Row 2 (Level 1) — Contains racks in that row as BOM children
      └— Rack C (Level 2) — Individual rack with component BOM

The Push POD Structure Wizard

Arena POD → BOM Operations → Push POD Structure to Arena

Loading Screen

The wizard starts by scanning your overview sheet and checking Arena for existing items (POD, rows, racks). This determines what needs to be created vs. what already exists.

Stage 1: Racks

1 Review and configure racks
  • Existing racks (already in Arena) are shown in green and are read-only — nothing to configure
  • New/placeholder racks require you to enter a name, description, and Arena category
  • A "bulk apply category" button lets you set the same category for all new racks at once

Stage 2: Rows

2 Review and configure rows
  • Rows are auto-generated from your overview's row layout
  • Enter a name and category for each row
  • Existing rows (from a previous push) are shown in green and skip validation

Stage 3: POD & Review

3 Configure the POD and review the full structure
  • Enter the POD name and select its Arena category
  • Review a full structure summary with a BOM tree preview showing all rows and racks
  • Click "Create in Arena" to begin the push

Creation in Progress

During creation, you'll see:

  • An animated progress bar with elapsed timer
  • A live step log showing each item being created
  • Rotating status quotes to keep you entertained while it works
Do not close the dialog or navigate away during the creation process. The wizard is making live API calls to Arena.

Completion

When the push finishes, the completion screen shows:

  • Total rack count and row count created
  • The POD item number assigned by Arena
  • An "Open POD in Arena" button that opens the Arena BOM viewer in a new tab

What Happens After a Push

After a successful push, your spreadsheet is automatically updated:

  • Cell A1 of the overview is updated with "POD: [name] ([NUMBER])"
  • Row Item column (Column B) is filled with clickable Arena part numbers for each row. The display text is the Arena part number and the link URL contains the item's GUID.
  • Rack statuses are updated to reflect their synced state
  • History events are logged for the push operation

Re-Pushing an Existing POD

If you've already pushed and want to update the structure:

  • The wizard detects existing POD, row, and rack items in Arena
  • Items that already exist are shown in green and skipped during creation
  • Only new or modified items are created/updated
  • This makes it safe to re-run the wizard after making changes to your layout
Tip: After pushing, click the Arena part number links in the overview's Row Item column to jump directly to those items in Arena PLM.

Consolidated BOM

Arena POD → BOM Operations → Create Consolidated BOM

While the POD push creates the hierarchy in Arena, the Consolidated BOM gives you a flat aggregated view of every component across all racks:

  1. Scans the overview sheet for all placed racks
  2. Reads each rack's BOM and aggregates quantities
  3. Creates a "Consolidated BOM" sheet with all items listed with total quantities
  4. A loading modal shows progress; a completion modal shows stats and links to the new sheet
Note: The Consolidated BOM sheet is recreated fresh each time (not updated incrementally). Run it again after any changes to get current totals.

Troubleshooting a Push

If you run into issues during a push:

  • Verify your Arena connection is working via Test Connection
  • Ensure BOM Levels are configured (categories for POD, Row, Rack types)
